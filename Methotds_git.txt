git init - инициализация папки , для того чтобы директорию подчинить под систему гита.
git clone -  клонировать проект
git add -  проинтедкировать файл
git commit -  сделать снимок проекта
git push - отправить изменения на удалённый репозиторий
git pull -  стянуть последние изменения на локальный репозиторий
git branch - конкретно эта команда показывает на какой ветке сейчас мы рбаотаем , но вот если -b , то создаёт и сразу перекидывает на созданную ветку
git checkout - переход с одной ветки на указанную ветку
git merge -  слияние двух веток , но есть отличе от рибэйза тем , что история гита после этой команды становиться как карта метрополитена большого города . если ей пользовались очень много рази много разработчиков на проекте
в общем , эта команда даёт полную фиксацию и эволюцию ветвей.
git rebase -  тоже слияние двух веток , но в отличие от мёрджа как бы выпримляет историю втетвей , переносит все коммиты в конеч истории ветки.
git cherry-pick -  эта команда переносит все коммиты в указунную втеку
git status -  отображает состояние файлов в системе гит
git log -  оторажает исотрию коммитов
git reset - противоположная команда git add , у git reset есть три вида команд --soft , --mixed ,  --hard
git revert - отмена коммита , но в истории будет виден коммит в котором это произошло git revert HEAD --no-edit
git stash - это команда помещает все изменения в буфер
git diff - используется для сравнение коммитов и файлов
